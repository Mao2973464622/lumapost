## 🚨 发送失败

### Q: 跑完了但邮箱没收到

**检查清单:**

1. 垃圾邮件箱（最常见）
2. `MAIL_PASS` 是 **授权码** 不是登录密码
3. QQ邮箱授权码获取: 设置→账户→POP3/IMAP/SMTP→开启→发送短信→获取16位授权码
4. 网易: 设置→POP3/SMTP/IMAP→开启→客户端授权密码

### Q: `❌ SMTP连接失败: Invalid login`

**原因**: 授权码错了。

**修复**:
1. 重新生成授权码
2. 确保复制时没多空格
3. `.env` 里不要加引号：`MAIL_PASS=abc123`（不是 `MAIL_PASS="abc123"`）

### Q: `❌ 邮件发送失败: 550 Mail content denied`

**原因**: QQ邮箱检测到"群发"特征，触发反垃圾。

**修复**:
1. 减少 `EMAIL_TO` 人数（< 5 人）
2. 把发件地址加白名单
3. 改用网易邮箱或 Resend

### Q: SMTP 可以测试通过但 GitHub Actions 失败

**原因**: GitHub Actions 的 IP 可能在某些邮箱的黑名单里。

**修复**: 改用 Resend API（免费 3000 封/月）。

```env
MAIL_PROVIDER=resend
MAIL_API_KEY=re_xxx
MAIL_FROM=noreply@yourdomain.com
```

## 🧠 AI 生成问题

### Q: `❌ 未配置 API Key`

**原因**: `.env` 或 GitHub Secrets 没有 `AI_API_KEY`。

**修复**:
1. 去 https://platform.deepseek.com 注册
2. API Keys → 创建 → 复制 sk-xxx
3. 本地: 写入 .env 的 `AI_API_KEY=sk-xxx`
4. GitHub: Settings → Secrets → 添加 `AI_API_KEY`

### Q: DeepSeek 余额不足

**现象**: `HTTP 402: insufficient balance`

**修复**:
1. 充 10 块钱够用半年（每天 4 次 ≈ 0.04 元）
2. 或换免费模型:

```env
# Google Gemini（每天 1500 次免费）
AI_PROVIDER=google
AI_API_KEY=your_google_key

# 通义千问（每天 100 万 token 免费）
AI_PROVIDER=qwen
AI_API_KEY=sk-xxx
AI_MODEL=qwen-turbo
```

### Q: AI 返回格式错误 / JSON 解析失败

**现象**: `⚠️ AI 返回非数组，使用降级数据`

**原因**: 有些 AI 模型不听话，在 JSON 外面包了 markdown 或解释文字。

**修复**: 试试换模型 `AI_MODEL=deepseek-chat`（DeepSeek 稳定性最高）

### Q: 4 个时段内容还是差不太多

**原因**: uapis.cn 热榜 24 小时变化不大。

**修复**: 改用 WorkBuddy 本地定时任务——它会用 WebSearch 实时搜不同关键词。或在 prompt 里加大差异化提示。

### Q: 能否用本地模型（不花 API 钱）

**能**。用 Ollama：

```env
AI_PROVIDER=custom
AI_BASE_URL=http://localhost:11434/v1
AI_MODEL=llama3:8b  # 或 qwen2.5:7b
AI_API_KEY=ollama
```

## 📮 邮件问题

### Q: 邮件在垃圾箱

**修复**:
- QQ邮箱: 设置→反垃圾→白名单→添加发件地址
- Gmail: 标记为"非垃圾邮件"几次后自动学习
- 网易: 同上

### Q: HTML 样式显示不对（Outlook）

**原因**: Outlook 2007-2019 用 Word 引擎渲染，不支持 CSS3 渐变和 flexbox。

**影响**: 渐变色变纯色、目录表格对齐不准、圆角变直角。

**当前状态**: 功能正常但不美观。计划加 VML fallback。

### Q: 邮件太大了（> 100KB）

v2.2 版本 16 版块邮件约 **130-150 KB**。如果邮箱限制 < 50KB：

```bash
# 只生成部分版块
ENABLED_SECTIONS=ai,hardware,finance,domestic,international,mobile npm run all:morning
```

### Q: 发多人时部分人收不到

**原因**: QQ 邮箱群发限制 20-50 人/次。

**修复**:
1. 减少 `EMAIL_TO` 人数
2. 改用 Resend（免费 3000 封/月，无群发限制）
3. 分批发送

### Q: 怎么换网易邮箱

```env
MAIL_PROVIDER=netease
MAIL_USER=your@163.com
MAIL_PASS=你的网易授权码
```

### Q: 怎么换 Gmail

1. 开启两步验证: https://myaccount.google.com/security
2. 生成应用专用密码: https://myaccount.google.com/apppasswords

```env
MAIL_PROVIDER=gmail
MAIL_USER=your@gmail.com
MAIL_PASS=应用专用密码（16位，无空格）
```

## ☁️ GitHub Actions 问题

### Q: Actions 一直 pending 不跑

**原因**: GitHub Free 版 Actions 有每月 2000 分钟限制，用完了就排队。

**检查**: 仓库 Settings → Actions → General → 查看使用量。

### Q: 手动 Run workflow 成功了但定时没跑

**原因**: GitHub Actions cron 不是精确的，可能有 5-15 分钟延迟。

```
cron: '0 22 * * *'  表示 UTC 22:00 左右触发（北京时间约 6:00-6:15）
```

### Q: `node: command not found`

**原因**: workflow 的 Setup Node.js 步骤跳过了。

**检查**: workflow 前几个 steps 是否都绿色 ✅

### Q: AI 步骤一直 pending 然后超时

**原因**: DeepSeek API 国内访问可能慢。

**修复**: 在 Actions workflow 里把 AI 超时加大：

```yaml
AI_TIMEOUT: '120000'
```

### Q: `Error: Cannot find module 'dotenv'`

**原因**: `npm install` 失败了（没装依赖）。

**检查**: workflow 日志里看 Install 步骤是否成功。

### Q: Artifact 太大了

每天 4 次 × 137KB = 548KB/天。GitHub 免费版 500MB 总限制，能用 1000 天。**不用管。**

### Q: 能不能在 GitHub 上只生成不发邮件

能。Run workflow → 勾选 **preview** → HTML 会作为附件上传供下载。

## 🐳 Docker 问题

### Q: docker compose up 报错

**常见原因**:

1. Docker Desktop 没启动
2. `.env` 文件不存在（`cp .env.example .env`）
3. 端口冲突

### Q: 容器内访问不了外网 API

**原因**: 国内 Docker Desktop 有时 DNS 有问题。

**修复**:
```yaml
# docker-compose.yml 加 DNS
services:
  lumapost:
    dns:
      - 114.114.114.114
      - 8.8.8.8
```

## 🔧 本地运行问题

### Q: Windows 上 Node.js 找不到

```powershell
# 检查是否安装
node --version
# 如果没有，去 https://nodejs.org 下载

# 如果装了但提示找不到，关闭重新打开 PowerShell/CMD
```

### Q: `npm install` 报错

```bash
# 清缓存
npm cache clean --force
# 删 node_modules 重装
rm -rf node_modules package-lock.json
npm install
```

### Q: `.env` 改了但无效

**原因**: `dotenv` 缓存。

**修复**:
```bash
# 确保 .env 在 lumapost/ 目录下
ls -la .env
# 重启终端
```

### Q: Windows 路径中文乱码

```powershell
# PowerShell 中
chcp 65001  # 切换到 UTF-8
```

## 📋 内容质量问题

### Q: 新闻看起来是编的/不够新

**可能原因**:
1. AI 在 hotboard 数据不足时会"脑补"——v2.1 已加降级提示
2. uapis.cn 热榜有 1-2 小时延迟
3. 深夜和早晨热榜更新慢

**提升方案**:
- 使用 WorkBuddy 本地任务（WebSearch 实时搜索）
- 加更多热榜源（微博热搜、百度热榜）

### Q: 理财版块没有数据

**原因**: 新浪股市 API 有时挂。

**影响**: stocks 数组为空，邮件会跳过股市数据显示区。不影响其他内容。

### Q: 游戏和动漫版块内容少

**原因**: uapis.cn 热榜里游戏动漫数据源较少。

**修复**: 在 prompt 里强调优先从 B站热榜 和 虎扑 挖掘相关内容。

## ⚡ 进阶排查

### 开启调试模式

```bash
# Node.js 调试
DEBUG=* node scripts/fetch-news-ai.js --period=evening

# 查看 AI 返回的原始内容
node scripts/ai-provider.js --test "测试"
```

### 手动测试每一步

```bash
# 1. 测试 AI
npm run ai:test

# 2. 测试邮件（发测试邮件）
npm run send:test

# 3. 只生成 JSON（不发邮件）
npm run fetch:evening

# 4. 只生成 HTML
npm run gen:evening

# 5. 只发送已有 HTML
npm run send:evening
```

### 查看 GitHub Actions 日志

仓库 → Actions → 点击失败的 run → 展开具体步骤 → 看红色报错。

### 重跑失败的任务

GitHub Actions → 点击失败的 run → 右上角 **Re-run jobs** → **Re-run all jobs**

## 💡 技巧

### 按兴趣筛选版块

```bash
# 只要 AI + 理财 + 汽车
ENABLED_SECTIONS=ai,finance,auto npm run all:morning

# 只要科技类（去掉游戏动漫体育影视）
ENABLED_SECTIONS=ai,hardware,github,mobile,auto,robot,space,internet npm run all:morning
```

### 定时只发工作日

修改 `.github/workflows/email.yml`:

```yaml
schedule:
  - cron: '0 22 * * 1-5'  # 周一到周五
```

### 自定义邮件主题

```env
MAIL_SUBJECT_PREFIX=📰 每日新闻 ·
```

或直接传参：

```bash
node scripts/mail-provider.js --send --subject="我的自定义标题" --body-file=data/email.html
```

### 批量测试所有版块配色

```bash
for period in morning noon afternoon evening; do
  npm run fetch:$period
  npm run gen:$period
done
# 4 个 HTML 都在 data/ 目录，打开看效果
```

### 监控 API 余额

DeepSeek: https://platform.deepseek.com → Usage → 看今日消耗

每天 4 次 × 约 4000 token ≈ **16000 token/天 ≈ ¥0.016/天 ≈ ¥0.48/月**
