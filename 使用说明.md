## 方式二：本地 Windows 运行

### 前提
- 安装 Node.js 18+：https://nodejs.org
- 下载项目：`git clone https://github.com/Mao2973464622/lumapost.git`

### 步骤

```powershell
cd C:\猫猫\项目\lumapost\lumapost

# 1. 安装依赖
npm install

# 2. 配置
copy .env.example .env
notepad .env   # 填入 AI_API_KEY / MAIL_USER / MAIL_PASS

# 3. 测试
npm run ai:test           # AI 能用吗
npm run send:test         # 邮件能发吗

# 4. 生成+发送早报
npm run all:morning       # 早报
npm run all:noon          # 午报
npm run all:afternoon     # 午后
npm run all:evening       # 晚间
```

### Windows 定时任务（关机不跑）

```powershell
# 创建4个计划任务
schtasks /create /tn "LumaPost-Morning"   /tr "npm run all:morning --prefix C:\path\lumapost"   /sc daily /st 06:00
schtasks /create /tn "LumaPost-Noon"      /tr "npm run all:noon --prefix C:\path\lumapost"      /sc daily /st 12:00
schtasks /create /tn "LumaPost-Afternoon" /tr "npm run all:afternoon --prefix C:\path\lumapost" /sc daily /st 16:00
schtasks /create /tn "LumaPost-Evening"   /tr "npm run all:evening --prefix C:\path\lumapost"   /sc daily /st 00:00
```

## 方式三：本地 Mac/Linux 运行

```bash
cd lumapost
npm install
cp .env.example .env
# vim .env 填入凭证

npm run all:morning
```

### crontab（关机不跑）

```bash
# 复制示例
cp crontab.example ~/lumapost-crontab
# 编辑路径
vim ~/lumapost-crontab
# 安装
crontab ~/lumapost-crontab
# 查看
crontab -l
```

## 方式四：Docker 部署

```bash

# 2. 配置 .env

# 3. 测试
docker compose run --rm lumapost all:morning

# 4. 启动定时器（每天4次自动发送）
docker compose up -d
```

## 方式五：VPS/服务器部署（systemd/crontab）

### 纯 crontab

```bash
# 安装 crontab
cp crontab.example /etc/cron.d/lumapost
# 编辑路径
vim /etc/cron.d/lumapost
```

### systemd + 定时器

```bash
# 复制服务文件
sudo cp lumapost.service /etc/systemd/system/
sudo cp lumapost-morning.timer /etc/systemd/system/
# 启动
sudo systemctl enable lumapost-morning.timer
sudo systemctl start lumapost-morning.timer
```

```bash

curl -X POST \
  -H "Authorization: token YOUR_GITHUB_TOKEN" \
  -H "Accept: application/vnd.github.v3+json" \
  https://api.github.com/repos/YOUR_USER/lumapost/dispatches \
  -d '{"event_type":"lumapost-send","client_payload":{"period":"morning"}}'
```

可用于：iOS 快捷指令 / Android Tasker / 其他自动化工具调用。

## 方式九：企业微信/钉钉 Webhook 通知

在 GitHub Actions workflow 最后加一步：

```yaml
- name: 通知企业微信
  if: always()
  run: |
    STATUS="${{ job.status }}"
    curl -X POST "YOUR_WEBHOOK_URL" \
      -H "Content-Type: application/json" \
      -d "{\"msgtype\":\"text\",\"text\":{\"content\":\"LumaPost $STATUS\"}}"
```

## 配置表

### AI 模型（在 .env 或 GitHub Secrets/Variables 设置）

| 变量 | 默认 | 说明 |
|------|------|------|
| `AI_PROVIDER` | `deepseek` | deepseek/minimax/openai/anthropic/google/qwen/zhipu/doubao/kimi/custom |
| `AI_API_KEY` | **必填** | API Key |
| `AI_MODEL` | 自动 | 模型名 |
| `AI_BASE_URL` | 自动 | 自定义 API 地址 |
| `AI_TEMP` | `0.7` | 创造性 0-1 |
| `AI_MAX_TOKENS` | `8000` | 最大输出 |

### 邮件（在 .env 或 GitHub Secrets 设置）

| 变量 | 默认 | 说明 |
|------|------|------|
| `MAIL_PROVIDER` | `qq` | qq/netease/netease-126/gmail/outlook/yahoo/aliyun/resend/sendgrid/mailgun/postmark/graph/custom-smtp |
| `MAIL_USER` | **必填** | 发件邮箱 |
| `MAIL_PASS` | **必填** | 授权码/API Key |
| `MAIL_FROM` | MAIL_USER | 发件人地址 |
| `MAIL_TO` | MAIL_USER | 收件人（多邮箱逗号分隔） |

### 其他

| 变量 | 默认 | 说明 |
|------|------|------|
| `WEATHER_LAT` | `28.20` | 天气纬度 |
| `WEATHER_LON` | `112.97` | 天气经度 |
| `WEATHER_CITY` | `湖南长沙` | 城市名 |
| `ENABLED_SECTIONS` | 全部 | 版块筛选（逗号分隔） |
